## 任务目标
在执行结果页面添加“修复测试用例”按钮。当测试执行失败（编译失败或用例未通过）时，用户点击该按钮可结合失败日志自动重新生成并执行测试用例。

## 方案设计

### 1. 后端接口增强
- **模型更新**：在 `app/models/testcase.py` 的 `GenerateTestRequest` 中增加 `failure_context` 可选字段，用于接收失败的日志信息。
- **路由逻辑**：在 `app/routers/testcase.py` 的 `generate_testcase` 接口中接收该字段并传递给 LLM 服务。
- **提示词优化**：在 `app/services/llm_service.py` 中，如果提供了 `failure_context`，则在 Prompt 中加入该错误信息，指示 AI 修复代码中的错误（如语法错误、断言失败或逻辑缺陷）。

### 2. 前端功能实现
- **按钮添加**：在 `Frontend/src/views/ResultView.vue` 的状态栏中增加“修复测试用例”按钮。
    - **显示条件**：当 `compile_success` 为 `false` 或 `test_result.failed > 0` 时显示。
- **修复流程**：
    1. 提取错误信息：如果是编译错误则提取 `stderr`，如果是运行失败则提取 `stdout`。
    2. 调用生成接口：发送包含错误信息的请求。
    3. 更新状态：生成成功后更新 Store 中的 `taskId`。
    4. 自动执行：紧接着调用 `startExecution` 触发新任务的编译与运行。
- **交互优化**：增加修复过程中的 Loading 状态，避免用户重复点击。

## 关键技术点
- **Prompt 工程**：确保 LLM 理解这是“修复”任务而非“初始生成”，需要重点关注失败信息中的具体位置和原因。
- **任务流衔接**：前端需要妥善处理任务 ID 的替换，确保执行页面能无缝过渡到新任务的结果展示。
