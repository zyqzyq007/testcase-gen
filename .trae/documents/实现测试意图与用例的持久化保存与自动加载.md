# 实现测试意图与用例的持久化保存与自动加载

为了解决“每次都需要重新生成”的问题，我将实现一套基于文件系统的缓存机制，将测试意图（Intent）和生成的测试用例关联到具体的函数上。

## 1. 后端修改 (Backend)

### 新增 `CacheService` 服务
*   创建 `app/services/cache_service.py`。
*   **存储位置**: `workspaces/_cache/<project_id>/<file_hash>_<func_name>.json`。
*   **键值策略**: 使用 `项目ID` + `文件路径` + `函数名` 作为唯一标识。相比使用行号，这种方式更能抵抗代码行号变动的影响。
*   **存储内容**:
    ```json
    {
      "intent": "生成的测试意图内容...",
      "latest_task_id": "关联的最新测试任务ID",
      "test_code": "生成的测试代码内容...",
      "updated_at": "时间戳"
    }
    ```

### 更新 API 逻辑
*   **修改 `POST /api/testcase/intent`**: 在意图生成完成后，调用 `CacheService` 进行保存。
*   **修改 `POST /api/testcase/generate`**: 在测试用例生成完成后，保存生成的代码和 Task ID 到缓存。
*   **新增 `POST /api/testcase/history`**: 
    *   接收 `project_id` 和 `function_id`。
    *   解析出函数名，查询缓存。
    *   返回历史数据（意图、代码、Task ID）。

## 2. 前端修改 (Frontend)

### 修改 `GenerateView.vue`
*   **加载时机**: 在 `fetchFunctionDetail` 或组件挂载（`onMounted`）时触发。
*   **逻辑**:
    1.  调用后端 `history` 接口。
    2.  如果存在历史 **Intent**，自动填充到输入框。
    3.  如果存在历史 **Test Code**，自动填充代码区域，并设置当前的 Task ID，以便用户可以直接点击“编译并执行”。

## 预期效果
当您再次进入之前已经生成过测试用例的函数页面时，之前的“测试意图”和“测试代码”会自动加载显示，无需重新生成。
