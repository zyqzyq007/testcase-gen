# 智能单元测试工具 · 输入输出说明

## 概述

智能单元测试工具读取 C/Python 语言源码工程，自动为每个被测函数生成单元测试代码，编译执行并采集覆盖率，最终输出每个函数的测试结果与生成的测试代码，供下游工具消费。本文档提供：

1. 数据卷约定
2. 输入格式
3. 输出格式（输出文件名、字段定义表、示例 JSON）

## 数据卷约定

文件夹命名为 `unit-test-generate`

集成模式下，源码工程从共享卷读取，目录结构为：

```
{portal_project_id}/{item_id}/{解压后的源码文件夹}/*.c, *.h
```

工具生成物（CPG、图谱、缓存、导出 JSON 等）写入本工具私有的可读写卷，不写回共享卷。

输出 JSON 写入共享卷路径：

```
{portal_project_id}/{item_id}/{解压后的源码文件夹}/unit-test-generate/unit-test-generate.json
```

## 输入格式

工具的输入是一个 **C 或 Python 语言源码工程**：单个源文件，或包含多个源文件的 `.zip` 压缩包。

| 字段 | 说明 | 是否必须 |
| :--- | :--- | :--- |
| `.c` / `.py` 文件 | 被测源码，工具自动从中解析函数作为测试目标 | 是 |
| `.h` 文件 | 头文件，作为生成测试时的上下文 | 否，建议提供 |
| `.json` 文件 | 设计文档（按函数名组织），用于第二步给测试代码插入中文注释 | 否 |

工具自动识别工程中"必测函数"（非 static 的、在头文件中声明的对外接口）作为测试目标。

### 上游工具输入（可选）

工具可自动读取同目录下上游工具的输出，为测试用例生成提供需求上下文：

| 上游工具 | 输出目录 | 文件 | 用途 |
| :--- | :--- | :--- | :--- |
| 文档审查工具 | `document-validator/` | `requirements.json` 或 `requirement.json` | 扁平化需求列表（含标题、正文、表格），两种文件名等价均支持 |
| 需求追溯工具 | `traceability_link_recovery/` | `traceability.json` | 需求↔函数映射（仅取 src_rank=1，每个函数的最佳需求） |

匹配规则：按函数名精确匹配 + 源文件 basename/后缀匹配。匹配成功后，LLM 生成 prompt 会追加需求标题、描述、表格约束，指导测试覆盖需求场景和边界条件。

两个上游输出均缺失时，按原有逻辑生成，无影响。

## 输出格式

工具输出一个 JSON 文件，记录工程内所有被测函数的测试结果与生成的测试代码。

**输出文件名**：`unit-test-generate.json`

### 字段定义表

| 字段路径 | 类型 | 必填？ | 说明 | 举例 |
| :--- | :--- | :--- | :--- | :--- |
| `project_id` | 字符串 | 是 | 工程唯一标识 | `"proj_20260608_a1b2c3"` |
| `project_name` | 字符串 | 是 | 工程名称 | `"MEMS陀螺"` |
| `exported_at` | 字符串 | 是 | 导出时间（北京时区 ISO8601） | `"2026-06-08T14:30:00.123456"` |
| `stats` | 对象 | 是 | 工程级测试结果统计 | 见下方 |
| `stats.total` | 整数 | 是 | 被测函数总数 | 12 |
| `stats.passed` | 整数 | 是 | 测试通过的函数数 | 9 |
| `stats.failed` | 整数 | 是 | 测试失败或被忽略的函数数 | 2 |
| `stats.compile_error` | 整数 | 是 | 编译失败的函数数 | 1 |
| `functions` | 数组 | 是 | 所有被测函数的测试结果，每个元素是一个对象 | 见下方 |
| `functions[].function_id` | 字符串 | 是 | 函数唯一标识（`base64(文件路径)_起始行号`） | `"a2V5LmM=_42"` |
| `functions[].name` | 字符串 | 是 | 函数名 | `"calc_wind_speed"` |
| `functions[].signature` | 字符串 | 是 | 函数签名 | `"int calc_wind_speed(float a, float b)"` |
| `functions[].source_file` | 字符串 | 是 | 函数所在源文件相对路径 | `"src/wind.c"` |
| `functions[].language` | 字符串 | 是 | 编程语言 | `"c"` 或 `"python"` |
| `functions[].task_id` | 字符串 | 否 | 最近一次测试任务 ID，为空表示尚未生成测试 | `"task_8f3a..."` |
| `functions[].last_run` | 字符串 | 否 | 最近一次执行时间 | `"2026-06-08T14:25:00"` |
| `functions[].compile_success` | 布尔 | 否 | 测试代码是否编译/安装成功 | `true` |
| `functions[].passed` | 整数 | 否 | 通过的测试用例数 | 5 |
| `functions[].failed` | 整数 | 否 | 失败的测试用例数 | 0 |
| `functions[].ignored` | 整数 | 否 | 被忽略的测试用例数 | 0 |
| `functions[].line_coverage` | 数字 | 否 | 行覆盖率（0-1） | 0.92 |
| `functions[].branch_coverage` | 数字 | 否 | 分支覆盖率（0-1） | 0.85 |
| `functions[].status` | 字符串 | 是 | 函数测试状态，枚举见下 | `"passed"` |
| `functions[].test_code` | 字符串 | 否 | 生成的测试代码 | `"#include \"unity.h\"\n..."` |
| `functions[].updated_at` | 字符串 | 否 | 该函数缓存最后更新时间 | `"2026-06-08T14:25:30"` |
| `functions[].compile_error_msg` | 字符串 | 否 | 编译失败时的具体错误信息（gcc/pip 输出） | `"Target compilation failed:\nsrc/wind.c:5: error..."` |
| `functions[].error_stage` | 字符串 | 否 | 编译失败的阶段 | `"target_compile"` / `"runner_compile"` / `"link"` / `"venv"` |
| `functions[].test_stdout` | 字符串 | 否 | 测试执行的标准输出（含断言失败详情） | `"test_runner.c:20:test_add_basic:PASS\n..."` |

**`status` 枚举值**：

| 值 | 含义 |
| :--- | :--- |
| `pending` | 尚未生成测试 |
| `compile_error` | 测试代码编译失败 |
| `failed` | 存在失败的测试用例 |
| `passed` | 测试全部通过 |
| `ignored` | 测试被忽略 |
| `no_tests` | 已生成但无可执行测试用例 |
