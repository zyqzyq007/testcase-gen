# 需求追溯工具

## **数据卷约定**

文件夹命名为`traceability-link-recovery`



## 输入格式：



需求：可基于文档验证工具的输出做调整，只需要req\_lable和content两个字段。

代码：支持代码文件直接输入。



## **输出格式：追溯链路**

### **文件名**



`traceability.json`



### **字段定义表**



|字段路径|类型|必填？|说明|举例|
|---|---|---|---|---|
|`metadata`|对象|是|追溯分析的元信息|见下方|
|`metadata.tool_name`|字符串|是|工具名称|`"traceability-tool"`|
|`metadata.tool_version`|字符串|是|工具版本号|`"2.0.0"`|
|`metadata.session_id`|字符串|是|会话唯一标识|`"a1b2c3d4-e5f6-..."`|
|`metadata.created_at`|字符串|是|生成时间（ISO 8601）|`"2026-06-08T14:30:00.000Z"`|
|`metadata.threshold`|数字|是|追溯链接的相似度阈值（仅保留 ≥ 此值）|`0.4`|
|`metadata.total_requirements`|整数|是|需求总数|`13`|
|`metadata.total_functions`|整数|是|切分出的函数总数|`47`|
|`metadata.total_links`|整数|是|追溯链接总数（≥ 阈值的）|`52`|
|`trace_links`|数组|是|追溯链接列表，按相似度降序排列|见下方|
|`trace_links[].requirement_label`|字符串|是|需求标识（与需求文件 `requirements.json` / `requirement.json` 的 key 对应）|`"XQ001 配置项初始化功能"`|
|`trace_links[].function_id`|字符串|是|函数 ID（与 `code_snippet.json` 中 `functions[].id` 对应）|`"func_012"`|
|`trace_links[].function_name`|字符串|是|函数名（便于直接阅读，无需查 code\_snippet\.json）|`"Notchl_Coeff_Init"`|
|`trace_links[].source_file`|字符串|是|函数所在源文件路径|`"codes/cj_buchang.c"`|
|`trace_links[].start_line`|整数|是|函数起始行号（1\-based）|`142`|
|`trace_links[].end_line`|整数|是|函数结束行号（1\-based）|`203`|
|`trace_links[].similarity`|数字|是|相似度得分（0\.0\-1\.0）|`0.8723`|
|`trace_links[].evidence`|字符串|否|追溯依据（LLM 判定理由，暂留空字符串，未来实现）|`""`|

## **输出格式：代码函数切片**



### **文件名**



`code_snippet.json`



### **字段定义表**



|字段路径|类型|必填？|说明|举例|
|---|---|---|---|---|
|`metadata`|对象|是|切分处理的元信息|见下方|
|`metadata.tool_name`|字符串|是|工具名称|`"traceability-tool"`|
|`metadata.tool_version`|字符串|是|工具版本号|`"2.0.0"`|
|`metadata.session_id`|字符串|是|会话唯一标识|`"a1b2c3d4-..."`|
|`metadata.created_at`|字符串|是|生成时间（ISO 8601）|`"2026-06-08T14:29:55.000Z"`|
|`metadata.total_files`|整数|是|源文件总数|`8`|
|`metadata.total_functions`|整数|是|切分出的函数总数|`47`|
|`metadata.language`|字符串|是|切分语言|`"c"`|
|`files`|数组|是|源文件列表|见下方|
|`files[].file_id`|字符串|是|文件唯一标识|`"file_001"`|
|`files[].path`|字符串|是|文件相对路径|`"codes/cj_buchang.c"`|
|`files[].total_lines`|整数|是|文件总行数|`520`|
|`files[].function_count`|整数|是|该文件切分出的函数数|`9`|
|`files[].functions`|数组|是|函数列表|见下方|
|`files[].functions[].id`|字符串|是|函数全局唯一标识（与 `traceability.json` 中 `function_id` 对应）|`"func_001"`|
|`files[].functions[].name`|字符串|是|函数名|`"Notchl_Coeff_Init"`|
|`files[].functions[].signature`|字符串|是|函数签名（返回类型 \+ 函数名 \+ 形参列表）|`"static void Notchl_Coeff_Init(void)"`|
|`files[].functions[].return_type`|字符串|否|返回类型|`"void"`|
|`files[].functions[].parameters`|字符串数组|否|形参列表|`[]`|
|`files[].functions[].start_line`|整数|是|函数起始行号（1\-based）|`142`|
|`files[].functions[].end_line`|整数|是|函数结束行号（1\-based）|`203`|
|`files[].functions[].line_count`|整数|是|函数体行数|`62`|
|`files[].functions[].body`|字符串|是|函数完整源码（含签名行）|`"static void Notchl_Coeff_Init(void)\n{\n\t..."`|
|`files[].functions[].calls`|字符串数组|否|函数内部调用的其他函数名|`["tanf"]`|
|`files[].functions[].called_by`|字符串数组|否|被哪些函数调用|`["Para_Init"]`|
|`files[].non_function_blocks`|数组|否|不属于任何函数的代码块|见下方|
|`files[].non_function_blocks[].type`|字符串|是|代码块类型|`"include"` / `"macro"` / `"global_var"` / `"struct_def"` / `"comment"`|
|`files[].non_function_blocks[].content`|字符串|是|代码块内容|`"#include \"cj_buchang.h\""`|
|`files[].non_function_blocks[].start_line`|整数|是|起始行号|`1`|
|`files[].non_function_blocks[].end_line`|整数|是|结束行号|`1`|
