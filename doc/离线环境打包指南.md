# Python 项目离线环境打包指南

本工具在**完全离线**的容器内跑 Python 测试，且会强制使用环境自带的 Python 版本（如 3.8），
不会回退到容器的默认 Python。因此用户上传 Python 项目时，需要附带一个用 **conda-pack** 打包
的离线环境包（`env.tar.gz`），里面包含解释器、标准库和所有依赖。

> ⚠️ **必须用 `conda pack` 打包，不能用 `python -m venv` 手搓**。
> 只有 conda-pack 会把 `bin/python` + `lib/pythonX.Y/stdlib` + `site-packages` 全部捆绑，
> 且 `bin/python` 用相对路径解析标准库，离线裸跑即可用。手搓 venv 会导致断链、缺 stdlib、
> C 扩展 `.so` 版本不匹配等一系列问题。

---

## 一、打包命令（推荐）

以 `python 3.8 + numpy` 为例，版本和依赖按你项目的实际情况替换：

```bash
# 1. 安装打包工具（只需做一次）
conda install -y -c conda-forge conda-pack

# 2. 创建专用环境，并装好项目所需的【全部】依赖
conda create -y -n myenv python=3.8
conda activate myenv
conda install -y numpy pytest coverage pytest-cov   # ← 项目所有依赖都装到这里
# 如有 pip 专属包：
# pip install <你的包>

# 3. 打成可重定位的离线包（关键参数：--force --ignore-missing-files）
conda pack -n myenv -o env.tar.gz --force --ignore-missing-files

# 4. 把 env.tar.gz 连同源码一起打包上传（见第二节）
```

---

## 二、如何上传

工具支持以下三种上传形态，**都会被正确识别**：

1. **【推荐】源码 + env.tar.gz 放进同一个 zip**：
   ```
   myproject.zip
   ├── vector_utils.py        # 你的源码
   ├── requirements.txt
   └── env.tar.gz             # conda-pack 打的离线环境
   ```

2. **源码打成 zip，env.tar.gz 单独直传**。

3. **直接传 env.tar.gz**（纯环境，源码另外上传）。

---

## 三、关键注意事项

| 项 | 说明 |
|----|------|
| **必须是 conda pack** | 不能是 venv / virtualenv 手搓。 |
| **依赖要装全** | 环境里必须预先装好 `pytest`、`coverage`、`pytest-cov`（测试框架要用）以及项目全部依赖。离线容器内无法联网补装。 |
| **`--ignore-missing-files`** | 规避部分包文件被 pip/clutter 占用导致的打包报错，实测必需。 |
| **打包时机** | 确认环境已完全装好（`conda-meta/` 已写满、没有 conda 进程在跑）后再打包，否则包内文件不全。 |
| **解释器版本** | 包内 `bin/python` 的版本即测试用的版本（如 3.8），与容器默认 Python 隔离，互不影响。 |

---

## 四、工具侧处理流程（供理解）

上传后工具会自动完成，无需人工干预：

1. 解压 `env.tar.gz` → 项目目录下的 `.conda_env/`（一次性，解完删原包）。
2. 测试时复用 `.conda_env/bin/python`，构造等价于 `source activate` 的环境变量
   （`PYTHONHOME` / `PYTHONPATH` / `PATH` 等），全程不联网。
3. `conda-unpack` 在整个环境生命周期内最多执行一次（用 `.unpacked` 标记守护，并发安全）。
4. `.conda_env` 目录会被函数解析和文件浏览**自动过滤**，不会污染项目函数列表 / 看板。
